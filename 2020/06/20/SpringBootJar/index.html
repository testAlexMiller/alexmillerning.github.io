  • MANIFEST.MF资源必须存放在/META-INF/目录下

JarLauncher的实现原理

java -jar命令实际上调用的方法

  • java -jar 命令执行时,/MAIN-INF/MANIFEST.MF中的Main-Class属性将调用其mian(String[])方法,实际上是调用JarLanuncher#launch(args)方法

解析JarLanuncher#launch(args)方法

代码

JarFile.registerUrlProtocolHandler()

前提概述
  • JDK默认支持文件(file),HTTP,JAR等协议
该方法的作用

个人理解是java -jar默认使用JDK内部的JAR协议,因此在启动Spring Boot时,如果使用java -jar命令,那么Spring Boot的jar包中的class包括lib都会因为路径不对而无法调用;
所以该方法扩展了JAR协议,以此可以使用java -jar来启动Spring Boot应用(又因为实现文件系统,也可以使用java 直接启动)

createClassLoader(getClassPathArchieves())

该方法获取前缀匹配BOOT-INF/classed或者BOOT-INF/lib的实例

launch(args, getMainClass(), classLoader)

引导类

结论

  • 无论是JAR还是WAR,读取Spring Boot启动类均来自/META-INF/MANIFEST.MF中的Start-Class属性,因此,JarLauncher实际上是同进程内调用Start-Class类的main方法,并且在启动前准备好Class Path
