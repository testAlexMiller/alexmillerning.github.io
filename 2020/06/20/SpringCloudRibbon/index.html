客户端负载均衡: Spring Cloud Ribbon

便于理解和学习,首先搭建ribbon相关的spring cloud项目

创建只有pom文件的空MAVEN父工程

在父工程下分别创建spring cloud模块spring-cloud-provider,spring-cloud-eureka,spring-cloud-consumer

在父工程的pom文件中添加spring boot,spring cloud依赖(该依赖是子模块都要用到的依赖)

引入父依赖

  • 在所有模块的pom文件中以<parent>标签方式引入父依赖

搭建服务注册中心

  1. 在spring-cloud-eureka模块的pom文件中引入eureka服务相关的依赖
  1. 在主类中开启eureka服务(注意导入的包是否正确)
  1. 配置application.yml文件

搭建服务提供者

  1. 在spring-cloud-provider模块的pom文件中引入eureka客户端相关的依赖
  1. 在主类中开启eureka客户端(注意导入的包是否正确)
  1. 添加服务方的controller(注意导入的包是否正确)
  1. 配置application.yml文件

搭建服务消费者

  1. 在spring-cloud-consumer模块的pom文件中引入eureka客户端,ribbon相关的依赖
  1. 在主类中开启eureka客户端
  1. 添加消费方的controller
  1. 配置application.yml

访问服务

  • 依次启动spring-cloud-eureka->spring-cloud-provider->spring-cloud-consumer
  • 使用[http://localhost:9000/ribbon-consumer]访问服务

RestTemplate详解

GET请求

  1. getForEntity()
  • getForEntity(String url, Class responseType, Object… uriVariables)
  • getForEntity(String url, Class responseType, Map<String, ?> uriVariables)
  • getForEntity(URI url, Class responseType)
  1. getForObject()
  • getForObject(String url, Class responseType, Object… uriVariables)
  • getForObject(String url, Class responseType, Map<String, ?> uriVariables)
  • getForObject(URI url, Class responseType)

POST请求

  1. postForEntity()
  • postForEntity(String url, Object request, responseType, Object… uriVariables)
  • postForEntity(String url, Object request, Class responseType, Map<String, ?> uriVariables)
  • postForEntity(URI url, Object request, Class responseType)

方法与上述的get方法类似,此处不再赘述,需要注意的是request参数,该参数可以是一个普通的对象,也可以是一个HttpEntity对象.如果是一个普通对象,而非HttpEntity对象,RestTemplate会将请求对象转换为一个HttpEntity对象来处理,其中Objec就是request的类型,request内容会被视做完整的body来处理;而如果request是一个HttpEntity对象,那个就会被当做一个完整的HTTP请求对象来处理,这个request中不仅包含了body的内容,也包含了header的内容.

  1. postForObject()
  • postForObject(String url, Object request, responseType, Object… uriVariables)
  • postForObject(String url, Object request, Class responseType, Map<String, ?> uriVariables)
  • postForObject(URI url, Object request, Class responseType)
  1. postForLocation()
  • postForLocation(String url, Object request, Object… uriVariables)
  • postForLocation(String url, Object request, Map<String, ?> uriVariables)
  • postForLocation(URI url, Object request)

由于postForLocation函数会返回新的资源的URI,该URI就相当于指定了返回类型,所以此方法实现的POST请求不需要指定responseType

PUT请求

  • put(String url, Object request, Object… uriVariables)
  • put(String url, Object request, Map<String, ?> uriVariables)
  • put(URI url, Object request)

put函数为void类型

DELETE请求

  • delete(String url, Object… uriVariables)
  • delete(String url, Map<String, ?> uriVariables)
  • delete(URI url

由于我们在进行REST请求的时候,通常将DELETE请求的唯一标识拼接在url中,所以DELETE请求也不需要request的body信息
